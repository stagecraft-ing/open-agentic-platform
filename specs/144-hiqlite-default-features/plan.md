# Implementation Plan: Hiqlite default-features hygiene

**Branch**: `144-hiqlite-default-features` | **Date**: 2026-05-10 | **Spec**: [`spec.md`](./spec.md)

## Summary

Three lines of `Cargo.toml`, one lockfile regeneration, and a CI
verification pass. The work is one PR scoped to two crate manifests
plus the resulting `crates/Cargo.lock` diff. No source code, schema,
runtime, or chart changes.

The load-bearing realignments:

1. **One missing flag is the root cause.** Adding `default-features =
   false` on `crates/orchestrator/Cargo.toml:20` stops Cargo from
   unifying upstream defaults across the `crates/` workspace.
   Everything else in this spec is a follow-on cleanup that becomes
   visible (or trivially correct) once that flag is in place.
2. **`auto-heal` becomes an explicit choice.** Inherited-by-default
   today; promoted to explicit on both crates so the choice survives
   future feature-list edits.
3. **Two redundant entries drop.** Orchestrator `dlock` is dead;
   axiomregent `cache` is transitively required and need not be listed.
4. **Lockfile is the verification surface.** The visible benefit is
   `cron` and `toml 1.1.2+spec-1.1.0` leaving the hiqlite deps block
   in `crates/Cargo.lock`. `platform/services/deployd-api-rs/Cargo.lock`
   is unchanged (separate workspace; already correct).

## Phase 1 — Manifest edits + lockfile regen

**Purpose**: Apply the three-file change.

**Sequence:**

1. **Edit `crates/orchestrator/Cargo.toml:20`** to:
   ```toml
   hiqlite = { version = "~0.13", default-features = false, features = ["sqlite", "listen_notify_local", "auto-heal"], optional = true }
   ```
2. **Edit `crates/axiomregent/Cargo.toml:38`** to:
   ```toml
   hiqlite = { version = "~0.13", default-features = false, features = ["sqlite", "dlock", "listen_notify_local", "auto-heal"] }
   ```
3. **Regenerate the lockfile.** Run `cargo check --manifest-path
   crates/orchestrator/Cargo.toml` once (or `cargo generate-lockfile
   --manifest-path crates/Cargo.toml` against the workspace root) —
   Cargo rewrites `crates/Cargo.lock` to reflect the new feature
   posture without compiling the workspace. Inspect the diff:
   - Removed: `cron` block; `toml` package at version
     `1.1.2+spec-1.1.0` (the `+spec-1.1.0` discriminator is the
     hiqlite-internal `toml` fork; the regular `toml` crate at a
     different version may stay if some other workspace member needs
     it, which is benign).
   - Retained: `futures-util` (kept by explicit `listen_notify_local`),
     `cryptr`, `s3-simple`, `deadpool`, `rusqlite`, etc.
   - If anything unexpected changes, halt and re-read the diff before
     proceeding.
4. **No source-file edits.** No `.rs` file changes are part of this
   phase.

**Phase 1 exit criterion:** the working tree carries the three-file
diff and nothing else.

## Test strategy

- **No new tests authored.** The existing CI surface around the
  orchestrator and axiomregent crates already exercises the
  `local-sqlite` build for orchestrator and the desktop hiqlite path
  for axiomregent.
- **Lockfile assertion is a manual diff inspection.** Optionally, a
  one-shot test could `grep` the post-regen lockfile for
  `name = "cron"` and `version = "1.1.2+spec-1.1.0"` and assert
  absence; but adding a test for a one-time hygiene change is
  overkill given the lockfile is itself the assertion artifact.
- **CI parity coverage** comes from `.github/workflows/ci-orchestrator.yml`
  and the `crates/` workspace pipelines. No new workflow needed.

## Risks and rollback

- **Cargo unification surprise.** Edge case: a member of the
  `crates/` workspace not in the audit's scope explicitly opts into
  `cache`/`backup`/`auto-heal`/`toml` via its own manifest. If so,
  the lockfile diff in Phase 1 will preserve those crates. Pre-flight
  search:
  `grep -rnE 'features.*=.*\[.*"(backup|s3|toml|cache)"' crates/*/Cargo.toml`
  — confirm there is no third party.
- **Desktop build regression.** `apps/desktop/src-tauri` depends on
  `axiomregent`. Build the desktop crate against the new feature
  posture as a smoke check (`cargo check` against
  `apps/desktop/src-tauri/Cargo.toml`).
- **Rollback** is `git revert` of the manifest edits and a fresh
  `cargo build` to restore the lockfile. No persistent state is
  touched; rollback is symmetric with the change.
